# V3 一期 — 技术架构与实现流程

> **版本**：v1（与 V3 PRD v3 对齐）
> **日期**：2026-05-27
> **范围**：5 模块（E 家庭 / J 长辈 / B 情报 / A1 深伪 / F 海外暗网）
> **基础**：在 V2 现有架构上增量演进，**不重构**

---

## 0. 设计原则

1. **不破坏 V2 线上**：所有新接口走 `/api/v3/*`，沿用 V2 已验证的 V2 接口隔离策略
2. **DB 加字段不删字段**：迁移分阶段（先加 → 启用 → 清理），可灰度
3. **第三方服务可降级**：每个外部依赖都有 failover / 主备 / 跳过
4. **数据最小化原则**：合规为先 — 触发者隐藏、原始数据 24h 销毁、密码不离设备
5. **客户端能本地做的不上服务端**：减少服务端负担和合规风险

### 2.2 E 家庭模块（5 张新表）

```sql
-- 家庭组
CREATE TABLE family_group (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
  owner_user_id TEXT NOT NULL REFERENCES "user"(id),
  name TEXT,
  invite_code TEXT UNIQUE,
  invite_code_expires_at TIMESTAMP,
  created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_family_group_owner ON family_group(owner_user_id);

-- 家庭成员
CREATE TABLE family_member (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
  group_id TEXT NOT NULL REFERENCES family_group(id) ON DELETE CASCADE,
  user_id TEXT NOT NULL REFERENCES "user"(id),
  role TEXT NOT NULL CHECK(role IN ('owner','guardian','ward')),
  share_query_results BOOLEAN NOT NULL DEFAULT TRUE,
  joined_at TIMESTAMP NOT NULL DEFAULT NOW(),
  UNIQUE(group_id, user_id)
);
CREATE INDEX idx_family_member_user ON family_member(user_id);

-- 用户每日活跃日志（用于关怀机制）
CREATE TABLE user_activity (
  user_id TEXT NOT NULL REFERENCES "user"(id),
  date DATE NOT NULL,
  active_count INTEGER NOT NULL DEFAULT 0,
  first_active_at TIMESTAMP,
  last_active_at TIMESTAMP,
  PRIMARY KEY(user_id, date)
);
CREATE INDEX idx_user_activity_date ON user_activity(date);

-- 关怀提醒发送记录（避免重复轰炸）
CREATE TABLE family_care_notice (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
  group_id TEXT NOT NULL REFERENCES family_group(id),
  inactive_user_id TEXT NOT NULL REFERENCES "user"(id),
  notified_user_ids JSONB NOT NULL,
  days_inactive INTEGER NOT NULL,
  channel TEXT NOT NULL CHECK(channel IN ('push','sms')),
  sent_at TIMESTAMP NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_family_care_notice_group_date ON family_care_notice(group_id, sent_at DESC);

-- 官方匿名广播消息
CREATE TABLE family_broadcast (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::TEXT,
  group_id TEXT NOT NULL REFERENCES family_group(id),
  triggered_by_user_id TEXT NOT NULL REFERENCES "user"(id),
    -- 服务端可见，前端永不返回（保护隐私）
  content_type TEXT NOT NULL CHECK(content_type IN ('phone','url','sms','voice')),
  content_hash TEXT NOT NULL,  -- 用于排重，sha256(normalized_content)
  content_display TEXT NOT NULL,  -- 显示给家人的内容（可脱敏）
  result_label TEXT NOT NULL CHECK(result_label IN ('scam','safe','unknown')),
  result_detail JSONB NOT NULL,  -- AI 检测特征 + 建议
  source TEXT NOT NULL CHECK(source IN ('auto_query','manual_share')),
  created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  -- 同家庭同内容当天唯一
  UNIQUE(group_id, content_hash, ((created_at)::DATE))
);
CREATE INDEX idx_family_broadcast_group_time ON family_broadcast(group_id, created_at DESC);
```

### 2.5 F 暗网监控（仅海外，2 张新表）

### 4.2 E — 关怀机制 cron

```
每天凌晨 1:00 触发 careCronJob
  │
  ├─ 扫描 user_activity，找出 (NOW - last_active_at) ≥ 2 day 的用户
  │
  ├─ 对每个用户：
  │    ├─ 查 family_group_id
  │    ├─ 查 family_care_notice 表，是否今日已发提醒
  │    │   ├─ 已发 → 跳过
  │    │   └─ 未发 → 继续
  │    ├─ 计算 days_inactive
  │    │   ├─ 2 天 → channel = 'push'
  │    │   └─ 3+ 天 → channel = ['push', 'sms']
  │    ├─ 取家庭其他成员 user_ids
  │    ├─ 发 APNs push
  │    ├─ 如 sms：调用阿里云短信 / Twilio（限家庭每天 1 条）
  │    └─ INSERT family_care_notice 记录
  │
  └─ 完成 → 等明天
```

## 5. 关键技术决策

### 5.1 短信网关选型

| 厂商 | 用途 | 单价 | 关键考量 |
|---|---|---|---|
| **阿里云短信** | 国内用户 | ¥0.045/条 | 工信部备案合规 + 国内送达率 99% |
| **Twilio** | 海外用户 | $0.0075-0.05/条 | 全球覆盖 + 双方同意法务清晰 |

**实现策略**：
- 服务端按 `user.region_code` 路由（CN 前缀走阿里云，其他走 Twilio）
- 短信模板必须提前**报备**（特别国内）
- 限频：单家庭每天 1 条短信（DB 唯一约束 + Redis lock）
- 失败重试 3 次，第 3 次失败转 push 兜底

### 5.2 第三方 AI Provider 选型 + Failover

```typescript
// AI Service 编排（伪代码，复用 V2 已有 failover 机制）
class DeepfakeService {
  async detect(audioUrl: string, userRegion: string) {
    const providers = userRegion.startsWith('CN')
      ? ['火山引擎', 'iFLYTEK备', 'doubao_self_train']
      : ['reality_defender', 'pindrop', 'resemble_detect'];

    for (const p of providers) {
      try {
        const result = await this[`call_${p}`](audioUrl);
        return { ...result, provider: p };
      } catch (err) {
        log.warn(`provider ${p} failed`, err);
        continue;  // 自动 failover
      }
    }
    throw new ServiceUnavailable('All deepfake providers down');
  }
}
```

## 6. 安全 & 合规

### 6.1 触发者隐私保护（E 模块核心）

**问题**：family_broadcast 表里有 triggered_by_user_id，但前端永远不应该看到。

**实现策略**：
1. **数据库层**：Prisma Schema 不向前端暴露此字段
2. **服务层**：`FamilyService.getBroadcasts()` 只 select 不含 triggered_by_user_id 的字段
3. **响应层**：DTO 转换层（NestJS class-transformer）强制 `@Exclude()` 此字段
4. **审计**：admin 后台可以看到该字段（用于客服查问题），但需要二次认证 + 操作日志

```typescript
// 前端 DTO
export class FamilyBroadcastDto {
  @Expose() id: string;
  @Expose() content_type: string;
  @Expose() content_display: string;
  @Expose() result_label: string;
  @Expose() result_detail: object;
  @Expose() created_at: Date;
  @Exclude() triggered_by_user_id: never;  // 强制排除
}
```

### 6.2 邮箱监控目标加密

```typescript
// 用户邮箱不应明文存
async addBreachTarget(userId: string, email: string) {
  const normalized = email.toLowerCase().trim();

  // 1. 用于 HIBP 查询的可逆 hash（带 per-user salt 避免彩虹表）
  const lookupHash = crypto.createHmac('sha256', userSalt(userId))
    .update(normalized).digest('hex');

  // 2. 用于展示的脱敏值（部分明文）
  const displayValue = maskEmail(normalized);  // e.g. "sa***@gmail.com"

  return prisma.breachTarget.create({
    data: {
      target_value_hash: bcrypt.hashSync(normalized, 10),
      target_value_lookup: lookupHash,
      // 注意：display_value 仅前端展示用，可不入库
    }
  });
}
```

## 9. 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| **DB 迁移失败导致 V2 接口异常** | 低 | 高 | 严格按"加字段不删字段"原则；每次迁移前 backup；migration 分 5 步上线 |
| **A1 模型准确率 < 80%** | 中 | 中 | 锁 2 个 provider 备选 + 200 样本基准测试；准确率不达预期延迟上线 |

| **短信成本超预算** | 低 | 中 | 限频 1 条/家庭/天硬约束 + 周报监控 |
| **触发者隐私意外泄露** | 低 | 极高 | DTO 强制 @Exclude + 日志脱敏 + 测试 case 覆盖 |
