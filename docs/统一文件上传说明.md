# 统一文件上传说明

Server 端统一文件上传模块：所有图片走 OSS + CDN，禁止本地存储。

---

## 一、接口

### 1. POST /api/upload/file（统一入口）

- **Content-Type:** multipart/form-data  
- **字段:**  
  - `file`: 图片文件（必填）  
  - `type`: 字符串枚举（必填），见下表  
- **鉴权:** 需登录，Header `Authorization: Bearer <token>`

| type        | OSS 目录      | 用途示例     |
|-------------|----------------|--------------|
| avatar      | avatar/        | 用户头像     |
| report      | reports/       | 举报附图     |
| screenshot  | screenshots/   | 截图/分析图  |
| case        | cases/         | 案例图       |
| knowledge   | knowledge/     | 知识库附图   |

**文件限制：**  
- 类型：image/jpeg、image/png、image/webp  
- 大小：≤10MB  

**返回：** `{ "url": "https://cdn.isitsafe.com/{folder}/{filename}" }`  
文件名规则：`{userId}-{timestamp}.{ext}`

### 2. POST /api/upload/avatar（保留）

- 行为与旧版一致，内部调用统一 `uploadFile(..., "avatar")`  
- 字段：`file`（multipart）  
- 限制：image/jpeg、image/png，≤5MB（保持兼容）  
- 返回：`{ "url": "https://cdn.isitsafe.com/avatar/xxx.png" }`

## 二、OSS 目录结构

```
avatar/
reports/
screenshots/
cases/
knowledge/
```

---

## 三、UploadService 逻辑

统一方法：`uploadFile(buffer, type, userId, mimeType, fileSize?)`

1. 校验 type 属于枚举  
2. 校验 mimeType 允许  
3. 可选校验 fileSize ≤ 10MB  
4. 生成 objectKey：`{folder}/{userId}-{timestamp}.{ext}`  
5. 上传 OSS  
6. 返回 CDN URL  

不写本地磁盘，仅通过 OSS 客户端上传。

---

## 四、禁止行为

- 不允许将上传文件保存到 Server 本地磁盘  
- 不允许破坏现有 POST /api/upload/avatar 接口  
- 不允许因上传逻辑修改用户资料（PUT /api/user/profile）逻辑  

---

## 五、使用方

- **iOS：** 头像、举报附图、截图等  
- **Admin：** 用户头像、案例/知识库配图等  
- **AI 模块 / 举报系统：** 需落图时先调本上传接口拿到 URL，再写入业务表
