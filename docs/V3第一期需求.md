# 模块 1：E — 家庭守护账号（v3 完全重写）

## 1. 模块定位

让 1 个用户带来全家网络。降低门槛（创建免费），增加粘性（关怀机制），强化品牌（官方匿名广播）。

### 🚨 关键定位修订（v3）
- ❌ **不是**「监控父母手机」
- ❌ **不是**「家庭付费包」
- ✅ **是**「免费家庭关系网络」+「关怀提醒」+「官方匿名广播」

## 2. 市场痛点
- 中老年最容易被骗，但子女远在外地无法及时知道
- 家人之间转发反诈链接被视为"小题大做"，可信度低
- 用户付费门槛高，"家庭包"反而劝退家庭组创建

## 3. 三大核心机制

### 机制 ① — 免费家庭组（关系网络）

| 规则 | 说明 |
|---|---|
| 创建/加入 | **完全免费**，任何用户可创建 |
| 容量 | 1 组最多 5 人 |
| 角色 | owner（创建者，1 人）+ guardian（监护人，多个）+ ward（被监护人，多个） |
| 邀请方式 | 微信分享卡片 / 短信邀请 / Universal Link |
| Pro 权益 | **不会因为加入家庭而获得任何 Pro 特权** —— 订阅按 V2 现有「5 次/天突破」逻辑 |

### 机制 ② — 关怀机制（100% 免费）

| 触发 | 动作 |
|---|---|
| 用户**真实主动**打开 App 任一页面 1 次 | 服务端标记 `last_active_at` 更新；今日已活跃 ✓ |
| 凌晨 0:00 仍未活跃 | 标记「昨日未活跃」（家庭页可见状态变 ⚠️），不发 push |
| 连续 2 天未活跃 | **自动 push** 家庭其他成员："X 已 2 天未活跃，去看看？" |
| 连续 3 天未活跃 | **push + 短信**（每家庭限 1 条短信/天，免费） |
| 用户重新打开 | 状态恢复 🟢，自动取消未发出的提醒 |

**关键定义**：
- 「活跃」 = 用户主动打开 App（点击图标 / 通知中心拉起后**进入主界面停留**）。**仅 push 被点击但无后续动作不算活跃**。
- 服务端区分活跃来源：客户端在用户每次冷启或回到 foreground 时调用 `POST /api/v3/user/heartbeat`，服务端记录时间戳。

**短信文案**：
- 中文："【StarLens】您的家人 [脱敏名] 已连续 3 天未打开 App，请联系确认安全。点此查看：xxx"
- 英文："[StarLens] Your family member [name] hasn't opened the app for 3 days. Please check on them: xxx"

### 机制 ③ — 官方匿名广播

#### 触发场景
| 入口 | 流程 |
|---|---|
| **自动**：家庭任何成员在 StarLens 内查询，结果为「高风险」 | AI 检测 → 已分类为「诈骗」 → 自动以「官方」名义广播给家庭所有人 |
| **手动**：成员主动「分享一条信息到家庭」 | 输入内容 → AI 先检测 → 按结果以「官方」名义广播 |

#### 三种结果形态

| 形态 | 颜色 | 文案 | 触发条件 |
|---|---|---|---|
| **诈骗** | 🔴 红 | "📢 StarLens 官方提醒：已识别诈骗" | AI 识别为 high_risk |
| **安全** | 🟢 绿 | "📢 StarLens 官方核实：暂未发现风险" | AI 识别为 low_risk |
| **未知** | 🟡 黄 | "📢 StarLens 官方提示：暂无法确认真伪，请谨慎对待" | AI 无法识别或 medium_risk |

#### 隐私约定
- **不显示触发者**：消息只显示「由家庭成员触发，已脱敏」
- 用户本人在「我的」可以看自己触发过多少次（不公开给家庭其他人）

#### 排重规则
- 同一条信息（同号码 / 同链接 / 同消息哈希）家庭内**一天内只发 1 次**
- 即使不同成员都触发同一内容，也只发 1 次

#### 配额
- 免费：**1 条官方提醒/家庭/天**（任意形态合计）
- 家庭 Pro：**不限**
- 当日免费配额用完后，AI 仍会检测但不发广播；用户在「家庭页 → 今日已拦截 N 条」可看到摘要，引导付费

#### 手动分享流程详细

```
1. 用户在「家庭」Tab 点「📤 分享一条信息给家人」
2. 弹出输入框：粘贴号码/链接/消息
3. AI 后台检测（≤ 5 秒）
4. 根据结果弹框：
   a. 识别为诈骗 → 弹框："将以官方名义发送给家人 ✓ 确认"
   b. 识别为安全 → 弹框："已核实为安全，是否仍以官方名义告知家人不必担心？" → 确认/取消
   c. 未识别 → 弹框："AI 暂无法判断，是否仍以官方名义告知家人保持警惕？" → 确认/取消
5. 用户确认后发送，家庭成员收到 push + 在「情报案例 Tab」可见
```

## 4. User Stories

### E-01 创建家庭组（免费）
> 任何用户都可创建家庭组，免费

### E-02 邀请家人
> 通过微信卡片 / 短信 / Universal Link 邀请，最多 5 人

### E-03 接受邀请
> 邀请方式：邀请码 6 位有效期 7 天 / 短链 Universal Link

### E-04 关怀活跃监控（自动 + 免费）
> 每个成员每天打开 App 自动更新活跃状态

### E-05 关怀提醒推送
> 连续 2 天未活跃 → push；连续 3 天 → push + 短信

### E-06 主动分享一条信息到家庭
> AI 检测 + 按结果弹框确认 + 官方匿名广播

### E-07 接收官方提醒
> 家庭其他成员收到「StarLens 官方」push + 在「情报案例」Tab 可见全部历史

### E-08 自动同步检测结果
> 任一家庭成员在 StarLens 查询 → 结果为高风险时自动官方广播（每天限 1 条免费）

### E-09 隐私控制
> ward 可关闭"我触发的查询触发官方广播"

### E-10 解散/退出家庭组
> owner 可解散；成员可退出

## 8. 数据模型

```sql
CREATE TABLE family_group (
  id TEXT PRIMARY KEY,
  owner_user_id TEXT NOT NULL,
  name TEXT,
  invite_code TEXT UNIQUE,
  invite_code_expires_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE family_member (
  id TEXT PRIMARY KEY,
  group_id TEXT REFERENCES family_group(id),
  user_id TEXT REFERENCES user(id),
  role TEXT NOT NULL,  -- 'owner' | 'guardian' | 'ward'
  share_query_results BOOL DEFAULT TRUE,  -- 是否允许查询结果触发官方广播
  joined_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(group_id, user_id)
);

-- 活跃日志（用于关怀机制）
CREATE TABLE user_activity (
  user_id TEXT REFERENCES user(id),
  date DATE NOT NULL,
  active_count INTEGER DEFAULT 0,
  first_active_at TIMESTAMP,
  last_active_at TIMESTAMP,
  PRIMARY KEY(user_id, date)
);

-- 关怀提醒发出记录
CREATE TABLE family_care_notice (
  id TEXT PRIMARY KEY,
  group_id TEXT REFERENCES family_group(id),
  inactive_user_id TEXT REFERENCES user(id),  -- 谁未活跃
  notified_user_ids JSONB NOT NULL,            -- 通知了哪些 guardian
  days_inactive INTEGER NOT NULL,
  channel TEXT NOT NULL,                       -- 'push' | 'sms'
  sent_at TIMESTAMP DEFAULT NOW()
);

-- 官方广播消息
CREATE TABLE family_broadcast (
  id TEXT PRIMARY KEY,
  group_id TEXT REFERENCES family_group(id),
  triggered_by_user_id TEXT REFERENCES user(id),  -- 仅服务端可见，前端不显示
  content_type TEXT NOT NULL,                     -- 'phone' | 'url' | 'sms' | 'voice'
  content_hash TEXT NOT NULL,                     -- 用于排重
  content_display TEXT NOT NULL,                  -- 显示给家人的内容（如脱敏号码）
  result_label TEXT NOT NULL,                     -- 'scam' | 'safe' | 'unknown'
  result_detail JSONB NOT NULL,                   -- AI 检测特征 + 建议
  source TEXT NOT NULL,                           -- 'auto_query' | 'manual_share'
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(group_id, content_hash, DATE(created_at)) -- 同家庭同内容当天唯一
);

CREATE INDEX idx_family_broadcast_group_time ON family_broadcast(group_id, created_at DESC);
```

CREATE TABLE intel_alert (
