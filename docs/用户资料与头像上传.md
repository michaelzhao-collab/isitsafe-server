# 用户资料与头像上传

本文档说明用户资料字段、头像上传流程及 Server / Admin / iOS 三端统一约定。

---

## 一、用户表字段（users）

在现有表上**仅新增**以下字段，不删除、不重命名已有字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| avatar | string (nullable) | 头像 CDN URL，如 `https://cdn.isitsafe.com/avatar/xxx.png` |
| nickname | string (nullable) | 昵称 |
| gender | string | 枚举：male / female / unknown，默认 unknown |
| birthday | datetime (nullable) | 生日，接口层使用 YYYY-MM-DD 字符串 |

---

## 二、接口约定（三端一致）

### 1. GET /api/auth/userinfo

返回中增加：`avatar`、`nickname`、`gender`、`birthday`、`subscriptionStatus`。  
- birthday 返回格式：`YYYY-MM-DD`  
- 头像仅返回 URL，通过 CDN 访问。

### 2. PUT /api/user/profile

请求体（JSON，字段可选）：

- `avatar`: string | null  
- `nickname`: string | null  
- `gender`: "male" | "female" | "unknown"  
- `birthday`: string | null（YYYY-MM-DD）

返回：`{ "success": true }`

### 3. POST /api/upload/avatar

- 请求：`multipart/form-data`，字段名 `file`（JPEG/PNG，建议 ≤5MB）  
- 服务端：接收文件 → 上传 OSS → 返回 CDN URL  
- 返回：`{ "url": "https://cdn.isitsafe.com/avatar/xxx.png" }`  
- 禁止在 Server 本地磁盘存储图片，一律走 OSS。

---

## 三、OSS 与 CDN

- **SDK：** Server 使用 `ali-oss`。  
- **路径规则：** `avatar/{userId}-{timestamp}.png`（或 .jpg）。  
- **访问：** 统一使用 CDN 域名，如 `https://cdn.isitsafe.com/avatar/xxx.png`。  
- **环境变量（Server）：**  
  - `OSS_REGION`、`OSS_BUCKET`、`OSS_ACCESS_KEY_ID`、`OSS_ACCESS_KEY_SECRET`  
  - `CDN_DOMAIN`（如 `https://cdn.isitsafe.com`）

---

## 四、Admin 后台

- **用户列表：** 展示头像（Avatar 组件）、昵称、手机、国家、性别、生日、注册时间、订阅状态。  
- **用户详情：** 展示并支持编辑 avatar、nickname、gender、birthday。  
- **接口：**  
  - GET /api/admin/users：列表含上述新字段。  
  - GET /api/admin/users/:id：单用户详情。  
  - PUT /api/admin/users/:id：编辑用户资料（同上字段）。

---

## 五、iOS 客户端

- **UserInfoResponse：** 增加 `avatar`、`nickname`、`gender`、`birthday`、`subscriptionStatus`。  
- **个人资料页：** 展示头像（CDN URL）、昵称、性别、生日；支持编辑。  
- **头像流程：**  
  1. 用户选择图片  
  2. 调用 POST /api/upload/avatar  
  3. 取得返回的 `url`  
  4. 调用 PUT /api/user/profile，传入 `avatar: url` 及其他需更新字段。

---

## 六、数据库迁移

在 Server 目录执行（需已配置 `DATABASE_URL`）：

```bash
npx prisma migrate dev --name add_user_profile_fields
```

将为 `users` 表新增 `avatar`、`nickname`、`gender`、`birthday` 列。
