# 🔍 V1 现状 ASO 诊断（2026-06-01）

| 维度 | 现状 | 评分 | 问题 |
|---|---|---|---|
| **App Name 关键词** | StarLens AI | 🔴 弱 | 只有品牌词，无场景/功能词，ASO 算法不给搜索流量 |
| **Subtitle 关键词** | Official / AI Risk Detection / Safety | 🟡 一般 | 缺 "scam / fraud / phishing / spam" 等高频搜索词 |
| **Description 第 1 屏** | 抽象介绍 "intelligent security assistant" | 🔴 弱 | App Store 只展示前 3 行，必须 Hook 用户痛点，不能像产品手册 |
| **关键词覆盖** | 约 6-8 个有效词 | 🔴 弱 | iPhone 用户搜 "scam detector" / "phishing check" / "anti-scam" 都搜不到我们 |
| **情感设计** | 偏理性、企业感 | 🟡 一般 | 反诈是高情绪场景（恐慌/愤怒/焦虑），文案需共情而非"官方" |
| **CTA / 用户分群** | 无 | 🔴 缺 | 没说"适合谁用"，用户犹豫 |
| **本地化** | 中国区无中文版 | 🔴 严重 | 中文区下载率会被砍 80% |

**核心问题**：当前文案是"产品视角"（我们做了什么），但 ASO 要求"用户视角"（用户解决什么具体问题）。

# ✅ V2 — 优化版（推荐使用）

## 设计原则

1. **App Name 必带 #1 关键词**（"Scam"）—— App Store 算法对名字字段加权最高
2. **Subtitle 写场景痛点 + 行动**（不是产品描述）
3. **Description 前 3 行 = 钩子**（用户看不到展开按钮，前 3 行决定下载与否）
4. **关键词字段塞满 100 字符**，不与 Name/Subtitle 重复
5. **中英双语独立优化**，中文不是英文翻译，要符合本地搜索习惯
6. **APP 主旨重新定位**：从"AI 风险检测工具"→"你的私人反诈助手 + 全家安全网"
