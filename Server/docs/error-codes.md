# 错误码说明（iOS 使用参考）

本文档整理通用 HTTP 状态码与建议的业务错误码，供 iOS 统一处理错误提示与重试逻辑。**当前 server 主要使用 HTTP 状态码 + message；业务错误码（如 10001）为约定格式，后续可在服务端统一返回。**

---

## 通用 HTTP 错误码

### 400 Bad Request（参数错误）

**HTTP statusCode：** 400  

**message 示例：**  
- `"Validation failed"`  
- `["content must be a string"]`（数组形式为 class-validator 校验详情）

**说明：**  
请求体或 Query 参数不符合校验规则（如必填项缺失、类型错误、格式错误）。

**iOS 建议提示文案：**  
- 通用：「请检查输入内容是否正确」  
- 分析/查询类：「输入内容有误，请检查后再试」  
- 举报类：「请完善举报类型和内容」

**是否可以重试：**  
可以。用户修正参数后重试。

---

### 401 Unauthorized（未登录 / Token 失效）

**HTTP statusCode：** 401  

**message 示例：**  
- `"Unauthorized"`  
- `"Invalid refresh token"`  
- `"User not found"`

**说明：**  
未携带 token、token 过期或无效、或 refreshToken 无效。部分接口（如登录、刷新 token）在凭证错误时也返回 401。

**iOS 建议提示文案：**  
- 需要登录的接口被 401：「登录已过期，请重新登录」  
- 登录接口 401：「手机号/邮箱或验证码错误」  
- 刷新 token 401：「登录已失效，请重新登录」

**是否可以重试：**  
- 普通接口 401：可先尝试用 refreshToken 换新 token，再重试原请求一次；若仍 401 则引导重新登录。  
- 登录/刷新 401：不需要自动重试，由用户重新输入或重新登录。

---

### 403 Forbidden（权限不足）

**HTTP statusCode：** 403  

**message 示例：**  
- `"Forbidden"`

**说明：**  
当前用户无权限访问该资源（如非 Admin 访问 admin 接口）。iOS 端不调用 admin 接口，一般不会遇到。

**iOS 建议提示文案：**  
「暂无权限访问」

**是否可以重试：**  
否。需更换有权限的账号或放弃该操作。

---

### 404 Not Found（资源不存在）

**HTTP statusCode：** 404  

**message 示例：**  
- `"Not Found"`  
- Prisma P2025 时可能为记录不存在

**说明：**  
请求的资源不存在（如知识库详情 ID 错误、不存在的用户等）。

**iOS 建议提示文案：**  
- 通用：「内容不存在或已下架」  
- 详情页：「该内容不存在或已下架」

**是否可以重试：**  
一般不需要。可返回上一页或刷新列表。

---

### 429 Too Many Requests（请求过于频繁）

**HTTP statusCode：** 429  

**message 示例：**  
- `"AI 分析请求过于频繁，请稍后再试"`  
- `"Account temporarily locked."`（登录失败次数过多）

**说明：**  
触发了服务端限流（如 AI 分析每分钟次数限制）或登录失败次数过多被临时锁定。

**iOS 建议提示文案：**  
- AI 分析：「操作过于频繁，请稍后再试」  
- 登录：「尝试次数过多，请稍后再试」

**是否可以重试：**  
可以，但需延迟。建议 429 后至少间隔 60 秒再允许用户重试，不要立即自动重试。

---

### 500 Internal Server Error（服务器错误）

**HTTP statusCode：** 500  

**message 示例：**  
- `"Internal Server Error"`  
- 或具体异常信息（生产环境可能被统一为通用文案）

**说明：**  
服务端未捕获的异常、数据库错误、第三方服务（如 AI）异常等。

**iOS 建议提示文案：**  
「服务暂时异常，请稍后重试」

**是否可以重试：**  
可以。可重试 1～2 次，间隔数秒；若持续 500 则提示稍后再试。

## 业务错误码（建议格式，后续可统一）

以下为建议的业务错误码与统一错误体格式，便于 iOS 做精细化提示。**当前 server 可能尚未全部返回 code 字段，iOS 可先按 HTTP 状态码 + message 处理，并预留对 code 的解析。**

**统一失败返回示例：**

```json
{
  "code": 10001,
  "message": "AI analysis failed",
  "detail": "模型调用失败或返回格式不正确"
}
```

---

### 10001 AI 分析失败

**message：**  
`AI analysis failed`（或中文：「AI 分析失败」）

**说明：**  
AI 模型调用失败、返回格式错误或服务端解析失败（如 JSON 校验不通过、超时、第三方 API 错误）。

**iOS 建议提示文案：**  
「分析失败，请稍后重试」

**是否可以重试：**  
是。建议最多重试 1 次，避免频繁调用产生成本或再次限流。

---

### 10002 OCR 识别失败

**message：**  
`OCR recognition failed`（或中文：「OCR 识别失败」）

**说明：**  
截图/图片的 OCR 识别失败（当前服务端 OCR 为预留，多数场景由客户端 OCR 后传文字，此码可在服务端未来实现 OCR 后使用）。

**iOS 建议提示文案：**  
「图片识别失败，请重试或手动输入文字」

**是否可以重试：**  
是。可重试或引导用户手动输入内容。

---

### 10003 订阅验证失败

**message：**  
`Subscription verification failed`（或中文：「订阅验证失败」）

**说明：**  
应用内购买收据验证失败（如 Apple 服务器校验未通过、收据无效或过期）。

**iOS 建议提示文案：**  
「订阅验证失败，请稍后重试或联系客服」

**是否可以重试：**  
是。用户可重试验证或联系客服。

---

### 10004 风险数据库查询失败

**message：**  
`Risk database query failed`（或中文：「风险库查询失败」）

**说明：**  
查询风险库（phone/url/company）时服务端异常（如数据库错误）。

**iOS 建议提示文案：**  
「查询失败，请稍后重试」

**是否可以重试：**  
是。可重试 1～2 次。

---

### 10005 知识库查询失败

**message：**  
`Knowledge base query failed`（或中文：「知识库查询失败」）

**说明：**  
获取知识库列表或详情时服务端异常（如数据库错误）。

**iOS 建议提示文案：**  
「加载失败，请稍后重试」

**是否可以重试：**  
是。可重试或下拉刷新。

---

## iOS 侧统一处理建议

1. **优先看 HTTP 状态码**  
   - 400：参数问题，提示用户检查输入。  
   - 401：token 问题，先尝试刷新 token 再重试，否则引导登录。  
   - 403：无权限，提示「暂无权限」。  
   - 404：资源不存在，提示「内容不存在或已下架」。  
   - 429：限流，提示「操作过于频繁」并延迟再试。  
   - 500：服务器异常，提示「服务暂时异常」并有限重试。

2. **若响应体中有 code 字段**  
   - 可解析为上述业务错误码，对应到更精准的提示文案。  
   - 未列出的 code 可统一用 `message` 或「操作失败，请稍后重试」。

3. **重试策略**  
   - GET、列表类：可重试 1～2 次。  
   - AI 分析：最多重试 1 次。  
   - 登录/支付/举报：不自动重试，由用户主动重试。  
   - 429：不立即重试，延迟（如 60 秒）后再允许操作。

4. **网络层错误**  
   - 无网络、超时、连接失败等：提示「网络异常，请稍后重试」，可重试。

本文档与当前 server 行为一致；若服务端后续增加并统一返回 code，以实际接口为准即可。
