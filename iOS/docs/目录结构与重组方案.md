# iOS 目录结构与重组方案

## 一、当前问题分析

### 1. 两套代码并存

| 位置 | 内容 | 是否被 Xcode 编译 |
|------|------|-------------------|
| **iOS/** 根下 | `App/`, `Views/`, `ViewModels/`, `Models/`, `Services/`, `Repositories/`, `Networking/`, `Storage/`, `Utils/`, `IAP/`, `Resources/` | **否**（没有 .xcodeproj 引用它们） |
| **iOS/IsItSafe1/IsItSafe1/** | `IsItSafe1App.swift`, `ContentView.swift`, `Theme/`, `Views/`, `Models/`, `Services/`, `Assets.xcassets` | **是**（当前唯一被编译的目录） |

- 工程文件在：`iOS/IsItSafe1/IsItSafe1.xcodeproj`
- 工程使用 **File System Synchronized Root**，只同步并编译 **`IsItSafe1/IsItSafe1/`** 这一层目录。
- 根下的 `App/`、`Views/`、`ViewModels/` 等是完整架构（MainTabView、HomeContainerView、接口层等），但**不在工程里**，所以 Xcode 从未编译过这套。

### 2. 同名双层嵌套

- 结构是：`iOS/IsItSafe1/`（工程所在）→ `IsItSafe1/`（同步根）→ 里面才是源码。
- 即「IsItSafe1 下面又有一个 IsItSafe1」，容易混淆，也和你期望的「iOS 下面整份代码放对」不一致。

### 3. 结论

- **真正在跑的是**：`IsItSafe1/IsItSafe1/` 里那套精简版（ContentView + 5 个 Tab + 简单 Services）。
- **你希望用的完整架构**：在 `iOS/` 根下的 App、Views、ViewModels、Networking 等，目前**没进工程**。

---

## 二、重组目标（怎么放最合理）

建议目标：

1. **只保留一套源码**：以完整架构（App、Views、ViewModels、Models、Services、Networking、Storage、Utils、IAP、Resources）为准。
2. **工程能编译这套**：让 Xcode 的同步根目录里就是这份完整代码，不再有两套重复。
3. **层级简单**：最好去掉「同名双层 IsItSafe1/IsItSafe1」，变成「工程 + 一个源码根目录」或「工程和源码同层」。

---

## 三、推荐方案：把完整代码移入当前同步根（不动工程位置）

**思路**：不移动 `.xcodeproj`，只把「完整架构」搬进当前 Xcode 正在同步的目录，并删掉里面的重复/旧文件，这样工程立刻编译到完整代码。

### 步骤概览

1. **备份**（建议）：复制一份 `iOS/IsItSafe1/` 或整份 `iOS/` 到别处。
2. **清空并替换** `iOS/IsItSafe1/IsItSafe1/` 内容：
   - 保留：`Assets.xcassets`、`Info.plist`（若在 IsItSafe1 层）。
   - 删除或移走：`IsItSafe1App.swift`、`ContentView.swift`、`Theme/`、`Views/`、`Models/`、`Services/`（当前 IsItSafe1 下的精简版）。
   - 把 `iOS/` 根下的这些目录/文件**移动**进 `iOS/IsItSafe1/IsItSafe1/`：  
     `App/`, `Views/`, `ViewModels/`, `Models/`, `Services/`, `Repositories/`, `Networking/`, `Storage/`, `Utils/`, `IAP/`, `Resources/`。
3. **入口与 target 名**：
   - 工程里主 target 仍是 IsItSafe1，入口需要是带 `@main` 的 App。把 `iOS/App/IsItSafeApp.swift` 移入 `IsItSafe1/IsItSafe1/App/` 后，需要让 target 的入口指向 `App/IsItSafeApp.swift`（若工程当前指向 `IsItSafe1App.swift`，可在 Xcode 里把入口改为 IsItSafeApp，或把 IsItSafeApp 内容合并进 IsItSafe1App 并改 module 名）。若使用「同步根」且保留 IsItSafe1 的 target 名，通常把 `IsItSafeApp.swift` 放到同步根下的 `App/` 即可，Xcode 会扫到 `@main`。
4. **Info.plist**：若在 `iOS/IsItSafe1/Info.plist`，保留不动；若在 `IsItSafe1/IsItSafe1/`，也保留在同步根内。
5. 在 Xcode 中 **Clean Build Folder**，再编译，按编译错误修 import / 模块名即可。

结果：  
- 目录仍是 `iOS/IsItSafe1/IsItSafe1/`，但里面是 **App、Views、ViewModels、Models…** 这一套，不再有同名两套代码。  
- 不再有「根下一大堆没被编译的代码」的困惑。

---

## 四、可选方案：工程上提到 iOS 根、去掉一层 IsItSafe1

若你希望结构变成「工程在 iOS 根、下面只有一个源码目录（如 IsItSafe）且不再双层」：

1. 把 `iOS/IsItSafe1/IsItSafe1.xcodeproj` **移动**到 `iOS/`。
2. 在 `iOS/` 下建一个目录作为「唯一源码根」，例如 `iOS/IsItSafe/`（或保留名 `IsItSafe1` 但只一层）。
3. 把当前 `iOS/` 下的 `App/`, `Views/`, … 全部移入 `iOS/IsItSafe/`；把 `iOS/IsItSafe1/IsItSafe1/Assets.xcassets`、`Info.plist` 等也移入 `iOS/IsItSafe/`。
4. 在 Xcode 中修改工程：
   - 把 target 的 **File System Synchronized Root** 从原来的 `IsItSafe1`（指向 `IsItSafe1/IsItSafe1`）改为指向 `IsItSafe`（即 `iOS/IsItSafe/`）。
   - 若有 `INFOPLIST_FILE` 等路径，改为相对新工程根的路径。
5. 删除或归档原来的 `iOS/IsItSafe1/IsItSafe1/` 和（若已搬空）`iOS/IsItSafe1/`。

这样最终结构类似：

```
iOS/
  IsItSafe1.xcodeproj   （或改名为 IsItSafe.xcodeproj）
  IsItSafe/             （仅一层，同步根）
    App/
    Views/
    ViewModels/
    Models/
    ...
  docs/
  README.md
```

---

## 五、建议优先做哪一步

- **想最少动工程、先跑起完整架构**：按 **第三部分** 做，把完整代码移入 `IsItSafe1/IsItSafe1/` 并替换旧内容。  
- **想顺带理清层级、去掉双层**：在第三步稳定编译通过后，再按 **第四部分** 把工程提到 `iOS/`、同步根改为单层 `IsItSafe/`。

如果你告诉我更倾向「只移代码进现有工程」还是「同时改工程位置 + 单层目录」，我可以按你选的方案写出更细的移动清单（含每个文件夹从哪里移到哪里）。
